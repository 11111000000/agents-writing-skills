# Tighten pass — 5 примеров (Lever 10 / Strunk cut-test)

> [!info] Контекст
> Tighten pass — Pass 2 Phase TIGHTEN в `humanize-writer` и `humanize-editor`. Применяет Grice submaxim 2: «Do not make your contribution more informative than is required» + Strunk cut-test + Williams 6 operations. Цель — сокращение 15-30% без потери смысла.

> [!warning] Bias substitution
> После каждого Tighten проверяйте, что **не потеряли факты** (числа, имена, команды). См. `humanize-editor` Pass 3 step 4.4.

Каждый пример содержит:
- Исходный текст (AI-typical)
- Tightened версию
- Word count diff
- Применённые scans (из 8)
- Сохранённые факты

---

## Пример 1: README проекта

### ❌ До (AI-typical, 56 слов)

> Добро пожаловать в наш проект! Это современное решение для управления задачами, которое поможет вам организовать рабочий процесс. Наш продукт сочетает в себе простоту использования, надёжность и широкие возможности настройки, что делает его оптимальным выбором для команд любого размера. Благодаря интуитивному интерфейсу и продуманной архитектуре, наш продукт позволяет сосредоточиться на главном.

### ✅ После (Tightened, 18 слов, −68%)

> **Genium Tasks** — CLI для тех, кому надоело вести задачи в Notion. Один бинарь, конфиг в `~/.config/genium/tasks.toml`, задачи живут в git вместе с кодом.

### Сохранённые факты

**Закономерности:**

2. **−60–77% reduction** — это не значит, что текст «плохой». Это значит, что LLM в 2-3× длиннее нужного.
3. **Burstiness σ растёт** — после Tighten pass короткие и длинные предложения чередуются естественно.
4. **Specificity вырастает с 0 до 3-4 фактов на абзац** — потому что удаляются абстракции, остаются числа и имена.

## Эвристики Tighten pass

1. **Удали первое предложение абзаца**, если оно общее. У 70% AI-текстов первое предложение вводное и не несёт информации.
2. **Удали последнее предложение**, если оно recap/вывод. У 60% AI-текстов последнее — повторение содержания.
3. **Замени три однородных на одно конкретное**. «Быстро, надёжно, безопасно» → «p99 14 мс, 99.99% uptime».
4. **Удали hedging конструкции**. «Возможно, в некоторых случаях...» → удалить целиком.
5. **Замени объяснение на факт**. «Это эффективное решение» → «время сократилось с 4 сек до 200 мс».
6. **Strunk cut-test**: удали любое предложение. Смысл выжил? Если да — удали.

## Где НЕ применять Tighten pass

- Техническая документация с шагами (пропущенный шаг = сломанная инструкция)
- Юридический текст (точность > стиль)
- Научный метод/результат (полнота обязательна)
- Tutorial для новичков (базовые шаги нужны)
- Переводы (сохранять оригинальную структуру)
