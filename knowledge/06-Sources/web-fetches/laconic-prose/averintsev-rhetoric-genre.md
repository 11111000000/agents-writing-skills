# Sergey Averintsev — Rhetoric, genre, and register

## TL;DR / Abstract

Averintsev is the guardrail for this repository. Russian brevity is powerful, but genre still rules the sentence. A README, an obituary, a legal note, and an essay cannot share the same rhythm. His work on rhetoric and literary tradition supports our boundary rule: apply Lever 12 only when the register allows it.

## Key points

No direct quotes here until a checked edition is added. This note anchors 4 claims:

1. Rhetoric is a historical system, not decoration.
2. Genre controls what counts as clear.
3. High style and short syntax can coexist.
4. Brevity can damage a text when the genre needs explicit framing.

## Connection to our work

- **When NOT to apply:** legal, onboarding, API reference, and beginner tutorials need more context.
- **Voice profile:** laconic mode should be opt-in for RU prose.
- **Bias substitution:** cutting text can remove genre obligations, not just extra words.
- **Audit output:** recommendations should name the register.

## Concrete application

Wrong rewrite for onboarding:

> Redis. Почему — ниже.

Better onboarding rewrite:

> Мы используем Redis для кеша сессий. Без него каждый запрос идёт в Postgres, и p95 растёт с 80 мс до 430 мс.

The second version stays explicit because the genre demands it.
