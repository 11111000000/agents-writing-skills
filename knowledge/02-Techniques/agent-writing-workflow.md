# Как агенту писать текст в реальном времени

> [!info] Контекст
> Когда агент пишет длинный текст, он не может «вспомнить» все правила сразу. Нужен практический алгоритм, встроенный в skill или в system-prompt. Этот документ — такой алгоритм.

## Алгоритм «Pre-flight → Write → Audit»

### Pre-flight (до написания)
1. **Определите голос**: 2–3 характеристики автора, аудиторию, цель.
2. **Соберите факты**: имена, числа, даты, ссылки. Без них не начинайте.
3. **Сформулируйте «крючок»** (lead) — первые 1–2 предложения должны быть конкретными.

### Write (само написание)
1. **Первый черновик** — пишите «плохо», не думая о стиле. Просто закройте содержание.
2. **Перечитать абзац за абзацем** и применить «Show, don't tell»: заменить абстракции числами.
3. **Перечитать ещё раз** и применить burstiness: чередовать длины предложений.
4. **Зачистить лексику** по спискам [[lexicon-ru|lexicon-ru]] / [[lexicon-en|lexicon-en]].
5. **Сломать симметрию** — добавить 1–2 предложения «не по формату».

### Audit (финальная проверка)
1. **Чек-лист ниже** — пройти его по каждому абзацу.
2. **Прочитать вслух** — если звучит как корпоративный пресс-релиз, переписать.
3. **Прогнать через детектор** — если он видит >40% AI, искать места с низкой burstiness.

## Чек-лист качества текста

> [ ] Первые 2 предложения содержат конкретику или провокацию.
> [ ] Длины предложений варьируются (от <5 до >25 слов).
> [ ] Длины абзацев варьируются (от 1 до 6+ предложений).
> [ ] Нет списков из 3 одинаковых по грамматике пунктов.
> [ ] Нет слов-маркеров AI (см. лексические словари).
> [ ] Есть хотя бы 3 конкретных числа/имени/даты.
> [ ] Есть хотя бы одно «я» или «мы» (если формат позволяет).
> [ ] Есть позиция / мнение / вывод.
> [ ] Нет мета-параграфов «таким образом, мы рассмотрели…».
> [ ] Заключение не повторяет вступление.
